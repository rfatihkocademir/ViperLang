# 🐍 ViperLang: A "Vibe Coding" Experiment

Hello there! Welcome to ViperLang. This is an unconventional, slightly wild, and entirely experimental laboratory where we step outside traditional software engineering rules.

### 🌟 How far can "Vibe Coding" go?

ViperLang emerged while chasing the answer to one simple question: **"How far can we push the limits of vibe coding (LLM/AI-driven development)?"**

If an LLM (Large Language Model) wanted to build its own programming language, what would it look like?
- What kind of keyword structure would it choose?
- How would it optimize code density?
- How would it design structures that it (and other AI agents) could best understand and manage?

ViperLang continues to evolve as a living answer to these questions. Every line of code here is a proof-of-concept of what happens when a developer and an AI catch a "vibe" and push the boundaries together.

**Note:** This is not a professional product; it's an exploration space. Everything can change, break, or be rewritten at any moment. That’s exactly where the fun lies!

### 🛠️ Let's Set It Up!

If you want to join this experiment, the setup is easy:

```bash
# 1. Clone the project
git clone https://github.com/rfatihkocademir/ViperLang.git

# 2. Enter the folder
cd ViperLang

# 3. Run the installation script
./install.sh
```

Now you can step into this AI-native world by typing `viper` in your terminal.

### 🐍 Your First Viper "Vibe"

```viper
pr("Hello Viper! 🐍")
pr("Ready to build the future with vibe coding?")
```

To run it:
```bash
viper hello.vp
```
